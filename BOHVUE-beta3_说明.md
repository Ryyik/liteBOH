# BOHVUE-beta3 功能说明书

## 1. 项目概览
BOHVUE-beta3 是基于 Vue 3 打造的 **方块之家 (BlockOfHome)** 官方 Web 平台的最新迭代版本。本项目旨在为社区成员提供集资讯、互动体验、周边商城及个人中心于一体的综合性数字化平台。

---

## 2. 目录结构指南

### 📂 /src (源代码目录)
项目的核心逻辑所在，采用模块化开发模式。

*   **📂 components/**: 通用 UI 组件。
    *   `UnifiedNavbar.vue`: 响应式统一导航栏，集成登录状态与移动端菜单。
    *   `Footer.vue`: 页面底部版权与辅助链接。
    *   `Overlay.vue`: 全屏加载/过渡动画组件。
    *   `UserCenterNav.vue`: 个人中心侧边导航组件。
*   **📂 views/**: 页面级视图组件。
    *   `Home.vue`: 平台首页，集成动态照片墙、LCP 优化后的英雄区及内容推荐。
    *   `Newsroom.vue`: 新闻资讯中心，支持分类筛选与模态框详情展示。
    *   `Shop.vue`: 周边商城，提供商品展示、分类查看及详细信息弹窗。
    *   `Activities.vue`: 活动中心入口，采用嵌套路由设计。
    *   `AnnualReport2025.vue`: 2025 年度报告，集成沉浸式动画体验。
    *   `UserCenter.vue`: 个人中心基座，承载用户信息、积分及订阅管理。
    *   `MBTI.vue` / `Health.vue` / `Game.vue`: 互动体验类功能页面。
*   **📂 router/**: 路由配置。
    *   `index.js`: 定义单页应用的所有路径跳转，包含嵌套路由与重定向逻辑。
*   **📂 store/**: 状态管理。
    *   `index.js`: 使用 Vuex/Pinia 模式管理全局状态（如用户信息、登录状态）。
*   **📂 composables/**: 逻辑复用模块。
    *   `useNews.js`: 封装新闻获取与处理逻辑。
*   **📂 utils/**: 工具类。
    *   `auth-manager.js`: 负责用户认证、本地存储与权限校验逻辑。
*   **📂 styles/**: 样式系统。
    *   `common/`: 基础动画、玻璃 UI 规范。
    *   `pages/`: 针对特定页面的私有样式表。
    *   `vendor/`: 第三方库样式及全局统一样式。

### 📂 /public (静态资源目录)
无需构建工具处理的原始资源。

*   **📂 annual-report-2025/**: 存放年度报告的独立静态 HTML 页面。
    *   `cartoon.html`: 年度报告的动画/漫画交互页。
    *   `best.html`: 年度精选内容展示。
*   **📂 assets/**: 全局静态图片、字体及基础 CSS。

## 3. 核心功能模块解析

### 3.1 首页 (Home)
*   **动态照片墙**: 响应式展示社区精彩瞬间，支持多终端适配。
*   **内容聚合**: 自动拉取最新的三条新闻资讯及推荐周边商品。
*   **性能优化**: 针对首屏图片采用 `eager` 优先加载，非关键区域图片实施 `lazy` 懒加载。

### 3.2 新闻中心 (Newsroom)
*   **多维度筛选**: 支持按分类实时过滤资讯。
*   **详情预览**: 点击新闻卡片即可通过全屏模态框查看完整内容，无需跳转页面。

### 3.3 周边商城 (Shop)
*   **商品展示**: 采用瀑布流布局，集成 WebP 格式支持以提升加载速度。
*   **交互细节**: 具备商品详情弹窗，展示高清大图与规格信息。

### 3.4 个人中心系统 (User Center)
*   **用户信息管理**: 查看并修改基础个人资料。
*   **积分与订阅**: 追踪社区积分变动，管理当前的订阅服务状态。
*   **周边信息**: 管理周边邮寄地址等必要信息。

### 3.5 互动与年度报告
*   **沉浸式报告**: 通过 `/annual-report-2025` 入口，用户可查收个性化的年度回顾。
*   **MBTI 测试**: 针对社区成员设计的性格测试互动。

## 4. 技术特性与优化

*   **全站懒加载策略**: 所有 `<img>` 标签均配置了 `loading="lazy"` 或动态加载逻辑，显著降低首屏流量消耗。
*   **玻璃 UI 设计 (Glassmorphism)**: 全面采用半透明、模糊背影的现代化 UI 规范，提升视觉高级感。
*   **响应式适配**: 深度优化移动端体验，包含独立的汉堡菜单导航与触摸友好型卡片布局。
*   **认证系统**: 集成本地化认证管理，支持登录状态持久化。

---

## 5. 开发者规范
*   **组件开发**: 优先使用 `<script setup>` 语法糖。
*   **样式管理**: 遵循 `styles/pages/` 下的模块化 CSS 命名规范，避免全局样式污染。
*   **数据流**: 页面级数据通过 `src/data/` 模拟或通过 `composables` 统一分发。
